### What passed (evidence)

- **A1**: `default-deny-all` + scoped DNS/scrape allows live in `gateway`, `identity`,
  `websites`; per-component netpols + `default-deny-ingress` in `argocd`; repo manifests
  match cluster state (`kubectl get netpol -A` vs `deploy/policies/network/`).
- **A2**: Trivy CI gate (CRITICAL blocks, first-party `deploy/` scope); all GitHub Actions
  SHA-pinned (checked every workflow, zero unpinned `uses:`); `disallow-latest-tag`
  Enforce; showcase image cosign-signed keyless in CI with in-workflow verify.
- **A3**: no workload ServiceAccount bound to `cluster-admin` (only kubeadm groups);
  Kyverno controllers use scoped roles + `view`; gateway-api plugin role is minimal.
- **A4**: all kaddy-authored workloads carry the full ADR-0301 set (`owner`, `service`,
  `part-of`, `managed-by`, `track`, `data-classification`, `business-criticality`):
  crossplane, dex, clubhouse, putting-green, mulligan/mulligan-bg. `require-kaddy-labels`
  and `restrict-data-classification` are **Enforce** with documented, namespace-scoped
  infra excludes. Offline `kyverno test`: 28/28 pass.
- **A5**: classification is queryable — a single label query returns
  `public` (clubhouse, putting-green, mulligan), `internal` (crossplane),
  `confidential` (dex).
- **A6**: gitleaks full-history scan clean (169 commits); all 3 secrets in
  `deploy/secrets/` are SOPS/age-encrypted (`ENC[` verified, `.sops.yaml` scopes
  `data|stringData`); no plaintext credential in any `kind: Secret` occurrence
  (all are secretRef/TLS references or an empty upstream notification stub).
- **TLS**: cert-manager issues from a local CA ClusterIssuer; live certificates Ready
  for argocd-server, dex, clubhouse, putting-green; Let's Encrypt issuers staged.

## Compliance mapping

The mandatory label contract (ADR-0301) operationalises NIS2/KRITIS-style asset
management: every kaddy-authored workload declares an owner, a service identity, and a
`data-classification` (public/internal/confidential), enforced at admission and queryable
in one command — i.e. asset inventory and criticality classification are properties of
the platform, not a spreadsheet. Gaps that would matter under such regimes are the
segmentation holes (F-01, F-04, F-05) and the unverified-in-cluster image signature
(F-02): the *controls exist* (default-deny pattern, keyless signing, Audit policy) but
coverage is not yet total. No P0 exposure found.
