# Evidence — E14-S03 Nix VM deploy: boot-to-serve PROVEN (ADR-0303) — 2026-07-19

**Story:** E14-S03 · **Change:** `openspec/changes/e14-nix-golden-images/` · **Status:** ✅ SERVE PROVEN (loop3)

## ✅ RESOLVED (2026-07-19 loop3) — the from-scratch NixOS image SERVES on gridscale

**Root cause (confirmed, not just narrowed):** the golden `Caddyfile` uses the global
`metrics { per_host }` block, recognised only by **caddy ≥ 2.9**. nixos-24.11 ships caddy **2.8.4**,
which rejects that block at parse time (`unrecognized global option: metrics`) — so `caddy run` exits
before binding any socket and the image serves nothing. The Ubuntu/Packer image installs caddy from
apt "stable" (≥ 2.9), which is why the byte-identical Caddyfile serves there. This is exactly one of
the "config bug" candidates the QEMU boot test (below) narrowed to.

**Fix (commit `485efa0`):** pin `services.caddy.package` to caddy 2.10.x via a dedicated
`nixpkgs-caddy = nixos-25.05` flake input (`nix/flake.nix` + `flake.lock`); `tests/smoke/e14-offline.sh`
validates the Caddyfile against that same caddy. CI `e14-nix-image` rebuilt the `.gz` green (run
`29680576417`, built exactly `485efa0`).

**Live re-prove (2026-07-19 loop3, torn down clean):** uploaded the corrected `.gz` (535 MB) to
`s3://kaddy-tfstate/marketplace/nix-golden.gz`; `tofu apply -replace` re-registered the `caddy-nix`
Marketplace app + import so gridscale re-materialised from the new `.gz` (new import
`2bfb228a-59d9-4ba7-b466-d7f5252c3a4b`, was `3aa9777e`); `hack/e14-s03-live-prove.sh` deployed a VM
from it on public IP `185.241.34.52`:

| Check | Result |
| --- | --- |
| `GET /` | **HTTP 200** ✅ |
| `GET /healthz` | **200** body `ok` ✅ |
| `GET :2019/metrics` | **200**, **158 `caddy_` series** ✅ |

Torn down ruthlessly (`server`/`storage`/`ipv4` each deleted; tenant re-verified **0** `kaddy-e14`
orphans). The single ephemeral VM cost ≈ €0.05.

## Diagnosis — narrowed by a local QEMU boot test (the discriminator)

gridscale's console is VNC (not API-readable), so to get boot-stage signal the fixed `.gz` was
**booted locally in QEMU (TCG) with serial capture** — `qemu-system-x86_64 -drive
file=nixos.img,if=virtio -device virtio-net-pci -serial file:… -netdev user,hostfwd=tcp::8080-:80`.
Result — the serial console shows a **full, successful boot**:

```console
SeaBIOS … Booting from Hard Disk …
GRUB loading. Welcome to GRUB!  Booting `NixOS'
<<< Welcome to NixOS 24.11.20250630.50ab793 (x86_64) - ttyS0 >>>
nixos login:
```

…yet `GET localhost:8080/` (hostfwd → guest :80) returned `000` throughout — **the image boots but
Caddy does not serve**, the *same* symptom as on gridscale. This is decisive:

1. **Firmware mismatch — RULED OUT.** gridscale default = i440fx/BIOS; the image is BIOS/MBR (`raw`). It
   also boots under QEMU SeaBIOS. `raw-efi` is *not* the fix.
2. **Image broken / doesn't boot — RULED OUT.** It reaches `nixos login:` locally (kernel + initrd +
   virtio + systemd + getty all fine — the qemu-guest/virtio fix works).
3. **`.gz` format / gridscale import mismatch — RULED OUT.** The *same* raw disk boots identically in
   local QEMU; a gridscale-specific import problem would not reproduce locally. (This was the prior
   leading hypothesis — the QEMU test refuted it, saving a wasted snapshot-export rebuild.)
4. **✅ ACTUAL CAUSE (narrowed): the image boots but Caddy/network doesn't serve — a NixOS *config* bug,
   locally reproducible.** The identical Caddyfile serves fine on the Ubuntu/Packer VM (E13-S05), so the
   gap is NixOS-specific: candidates are (a) `services.caddy` under `admin off` not coming up / not
   binding :80 (Type=notify readiness, cap_net_bind), (b) the `/srv` `tmpfiles` symlink not ready when
   Caddy starts, or (c) `networking.useDHCP` not bringing the NIC up (no IP → unreachable on both QEMU
   hostfwd and the gridscale public IP). **Fix path (local, fast, no gridscale cost):** rebuild a
   *debuggable* image (root password + `systemctl status caddy` / `ip -br a` / `journalctl -u caddy` to
   ttyS0), boot in QEMU, read the exact failure, fix the module, then re-prove on gridscale.
